# 我用AI打造了一个工业级Prompt生成器，让每个人都能成为Prompt工程师

借知乎 #AI上新 和#用AI重新看见人 的话题，分享我如何用AI开发一个专业的Prompt工程工具。这个项目让普通人也能写出高质量Prompt，真正"看见"AI对话中的逻辑与质量。

## 一、为什么做这个项目？

### 1.1 Prompt质量决定AI对话成败

在大模型时代，Prompt质量直接影响AI输出效果。但很多人面临这些问题：

- **不会写Prompt**：只知道简单提问，不知道如何结构化指令
- **质量不稳定**：有时效果好，有时完全失败，找不到规律
- **缺乏反馈**：不知道自己的Prompt哪里有问题，无法改进
- **学习成本高**：看教程学理论，但实际应用还是不会

我看到太多人在使用AI时，因为Prompt质量差而得出错误的结论，甚至认为"AI不行"。实际上，问题不在AI，在于Prompt工程的专业性门槛。

### 1.2 痛点：从需求到高质量Prompt的鸿沟

举个例子，用户想做一个"数据分析助手"，可能会写：

```
帮我做一个数据分析助手
```

这是典型的低质量Prompt，缺少角色定义、任务说明、输出格式、约束规则等核心要素，AI只能给出泛泛的回答。

而专业的Prompt应该包含：

```markdown
你是一位资深数据分析师，擅长SQL、统计学、数据可视化...

任务：分析用户提供的数据，输出洞察和建议...

约束：
- 每次分析不超过500字
- 必须提供至少3个可执行的优化建议
- 使用专业术语但配通俗易懂的解释

输出格式：
【数据概况】...
【核心洞察】...
【优化建议】...
```

问题在于：普通人不知道这个结构，也不理解为什么需要这些要素。

**我想做的事：把这个专业过程自动化，让每个人都能生成高质量Prompt。**

## 二、产品介绍

### 2.1 工业级Prompt工程流水线

这个工具实现了完整的"解析 → 检测 → 生成 → 评分 → 优化"闭环流程，就像工厂的质检流水线：

![流程图](https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt=工业流程图：5个步骤的流水线，从需求解析到质量评分，青色科技风格，简洁商务风格，竖版布局，9:16&image_size=portrait_16_9)

**核心5步流程：**

1. **需求解析** - 从用户自然语言提取结构化信息（任务类型、角色、目标、格式等）
2. **缺口检测** - 自动识别信息缺失，生成精准澄清问题
3. **Prompt生成** - 基于Prompt工程最佳实践，生成完整Prompt
4. **质量评分** - 5维度评分系统（清晰度、完整性、可执行性、格式、安全性）
5. **自动优化** - 闭环优化机制，最多3次迭代确保达标

### 2.2 智能澄清系统

当用户输入需求后，系统会自动检测缺失信息：

![澄清界面](https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt=简洁的界面：三个独立的问题卡片，青色科技风格，玻璃卡片效果，金色标题，竖版布局&image_size=portrait_16_9)

系统会提出不超过3个精准问题，例如：

- "关于'全部改为中文公式'，是否包含数学符号（如 +, -, =）？"
- "输出格式希望是JSON还是Markdown？"
- "目标受众是程序员还是普通用户？"

用户只需选择或填写，系统自动补充完整信息。

### 2.3 实时质量评估

生成的Prompt会自动评分：

![评分界面](https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt=评分界面：5个维度柱状图，清晰度、完整性、可执行性、格式规范、安全性，青色渐变，竖版布局&image_size=portrait_16_9)

**5维评分标准：**

| 维度 | 标准 | 说明 |
|------|------|------|
| **清晰度** | 1-5分 | 指令是否无歧义，是否使用量化标准 |
| **完整性** | 1-5分 | 是否覆盖角色/任务/上下文/格式/约束 |
| **可执行性** | 1-5分 | AI能否直接照做，无需猜测 |
| **格式规范** | 1-5分 | 输出格式是否明确，是否提供模板 |
| **安全性** | 1-5分 | 是否有防止越界的约束 |

### 2.4 自动优化闭环

如果评分不达标（任一维度<4分），系统会自动优化：

![优化流程](https://trae-api-cn.mchost.guru/api/ide/v1/text_to_image?prompt=闭环优化流程图：生成-评分-优化-再评分，最多3次循环，青色科技风格，简洁商务风格&image_size=portrait_16_9)

**优化机制：**

- 提取评分建议（如"缺少明确输出格式"）
- AI自动改进Prompt（补充缺失要素）
- 再次评分验证效果
- 最多尝试3次，确保质量达标

## 三、开发过程：AI如何帮助我

### 3.1 全程AI辅助开发

这是一个有趣的悖论：**我用AI来开发一个帮助人写Prompt的工具**。

开发过程中，每个环节都依赖AI：

**技术栈选择：**

- 用Claude/GPT讨论架构设计
- AI推荐React + TypeScript + Vite组合
- 自动生成基础组件代码

**Prompt设计：**

这是最核心的部分，所有Prompt都使用XML标签隔离，符合大模型最佳实践：

```xml
<role>你是一位资深Prompt工程师...</role>

<task>根据结构化需求产出高质量Prompt...</task>

<generation_rules>
1. 必须包含以下7个部分...
2. 指令必须具体到可执行...
</generation_rules>
```

**为什么用XML标签？**

经过多次测试发现：
- XML标签让AI更清晰理解指令边界
- 减少"混淆"或"误解"的可能性
- 输出更稳定、格式更规范

### 3.2 关键技术挑战

**挑战1：澄清问题的答案同步问题**

最初用户填写多个澄清问题时，发现答案会"同步"（填写第一个，其他也显示相同内容）。

问题根源：React状态管理中，多个输入框共享同一对象引用。

解决方案：使用数组存储答案，每个索引对应唯一问题：

```typescript
const [clarificationAnswersArray, setClarificationAnswersArray] = useState<string[]>([]);

const updateClarificationAnswer = (index: number, value: string) => {
  setClarificationAnswersArray(prev => {
    const newAnswers = [...prev];
    newAnswers[index] = value; // 索引定位，绝对独立
    return newAnswers;
  });
};
```

**挑战2：评分字段名不匹配**

评分全是5分却显示"未达标"，原因是JSON字段名不一致：

- Prompt输出 `"pass": true`
- TypeScript类型 `passed: boolean`

导致解析后字段缺失，UI判断失败。

解决：统一字段名为 `"passed"`，确保JSON解析正确。

### 3.3 AI辅助调试

最神奇的是：**用AI来调试AI相关代码**。

遇到"复制无反馈"问题时，我问AI：

```
"用户点击复制按钮后，没有任何反馈提示，
需要添加成功提示，按钮文本变化，2秒后自动消失"
```

AI不仅给出代码方案，还解释了用户体验设计要点：
- 立即反馈（按钮文本变化）
- 视觉强化（青色提示框）
- 自动消失（不打扰用户）

## 四、核心创新：重新看见Prompt质量

### 4.1 "看见"隐藏的质量问题

这个工具最大的价值是让用户"看见"自己Prompt的问题：

**案例：用户输入"帮我分析销售数据"**

**原始Prompt（用户写的）：**
```
帮我分析销售数据
```

**系统生成的Prompt：**
```markdown
你是一位资深商业数据分析师，擅长零售行业数据分析...

【任务】
分析用户提供的销售数据，输出商业洞察和优化建议...

【输出格式】
### 数据概况
- 数据量、时间范围、核心指标

### 核心洞察
- 3个重要发现（增长率、异常点、趋势）

### 优化建议
- 每个建议包含：问题、原因、解决方案、预期效果

【约束】
- 分析不超过500字
- 必须提供至少3个可执行建议
- 使用专业术语但配通俗解释
```

**评分对比：**

| 维度 | 原始Prompt | 生成Prompt | 提升 |
|------|-----------|-----------|------|
| 清晰度 | 2/5 | 5/5 | +150% |
| 完整性 | 1/5 | 5/5 | +400% |
| 可执行性 | 2/5 | 5/5 | +150% |
| 格式规范 | 1/5 | 5/5 | +400% |
| 安全性 | 2/5 | 4/5 | +100% |

用户从此"看见"了专业Prompt的标准，理解了为什么需要结构化。

### 4.2 从"试错"到"科学"

传统方式：写Prompt → 试运行 → 不满意 → 改改改 → 还不满意 → 不知道怎么办

现在的方式：
1. 输入需求 → 系统解析结构
2. 回答问题 → 补充缺失信息
3. 生成Prompt → 自动评分
4. 查看建议 → 理解改进方向
5. 自动优化 → 确保质量达标

**从盲目的试错，变成有反馈的科学流程。**

### 4.3 降低专业门槛

Prompt工程的核心知识（角色、任务、格式、约束、示例），普通人不知道，也不理解。

这个工具通过：
- **自动提取**需求中的关键信息
- **主动澄清**缺失要素
- **实时评分**显示质量水平
- **具体建议**指明改进方向

让每个人都能写出专业级Prompt，真正降低使用门槛。
